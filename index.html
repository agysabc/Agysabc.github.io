Distributed storage

VedaCoin The entire data ledger is copied and stored on all nodes running VEDA. Unilateral modifications will not take effect. This mechanism guarantees that the VEDA network will not be controlled by one party..

Decentralization

UTXO model

UTXOU nspent transaction output: The input of each transaction must be the unspent output of a transaction, and each input also requires the private key corresponding to the previous output to be signed..

Transaction model

Asymmetric encryption

Public key and private key, that is, public key encryption and private key decryption. For example, when A sends information to B, A uses B's public key to encrypt the information. After B receives this message, B uses B's private key to decrypt A's message..

Safety

Longest chain mechanism

Network latency and hash generation are random. The possibility of two miners digging out a block at the same time will cause a fork of the blockchain.At this time, the system will choose according to the longest chain mechanism..
